## Steps

### 1. Prompt Copilot to generate INSERT statements

Open `src/main/resources/data.sql` in your editor, place your cursor at the end of the file, and ask Copilot:

> "Add INSERT statements to data.sql for 5 billboard advertising campaigns with realistic OutFront Media data. Include campaigns in different statuses (DRAFT, ACTIVE, COMPLETED, CANCELLED). Use locations like Times Square, Sunset Boulevard, etc."

### 2. Review the generated SQL

Check the INSERT statements against these conventions:

- [ ] **UPPER_CASE SQL keywords:** `INSERT INTO`, `VALUES`
- [ ] **snake_case column names:** `start_date`, `end_date`, `created_at` тАФ not `startDate`
- [ ] **Date format:** `'2024-11-01'` (ISO format, single-quoted)
- [ ] **Enum values as strings:** `'ACTIVE'`, `'DRAFT'` тАФ matching the `CampaignStatus` enum names
- [ ] **Budget as a number:** `50000.00` тАФ not a string
- [ ] **All statuses represented:** At least DRAFT, ACTIVE, and COMPLETED across the 5 rows
- [ ] **Table name matches `@Table` annotation:** `campaign` (not `campaigns`)

> ЁЯТб **Tip:** The SQL must be compatible with both H2 and SQL Server. Avoid H2-only or SQL Server-only syntax. Standard SQL works in both.

### 3. Check column order matches the entity

The INSERT columns should correspond to `Campaign.java` fields. If your entity has these fields:

```
name, client, start_date, end_date, budget, status, location, created_at
```

Then the INSERT should list columns in the same order (or any consistent order).

> тЪая╕П **Don't include the `id` column** тАФ it's auto-generated by `@GeneratedValue`.

### 4. Restart the app and verify

Stop the running app (Ctrl+C) and restart:

```bash
mvn spring-boot:run
```

Watch the console output. If the INSERT statements have errors, you'll see SQL exceptions during startup. Fix any issues Copilot suggests.

### 5. Query campaigns in the H2 console

1. Open **http://localhost:8080/h2-console**
2. Connect with JDBC URL `jdbc:h2:mem:omsdb`, user `sa`, no password
3. Run:

```sql
SELECT id, name, client, status, budget FROM campaign;
```

You should see 5 rows with different statuses and realistic campaign data.

Try a filtered query to preview what the repository methods will do:

```sql
SELECT name, client, status FROM campaign WHERE status = 'ACTIVE';
```
