### Key Findings

**Codebase Patterns (Consistency Alignment):**
- Inner enum pattern for status fields (`CampaignStatus` mirrors `Order.OrderStatus`)
- `@GeneratedValue(strategy = GenerationType.IDENTITY)` for all PKs
- Constructor injection exclusively (no field `@Autowired`)
- `LocalDate` used for timestamps (not `LocalDateTime`)
- `double` used for `InventoryItem.unitPrice` (flagged as tech debt; Campaign uses `BigDecimal`)
- GlobalExceptionHandler handles: `ResourceNotFoundException`, `InsufficientStockException`, `IllegalStateException`
- Entities returned directly from controllers (no DTOs yet)
- Seed data in `data.sql`: plain INSERT statements, grouped by entity

**Architecture Decisions Made:**
- **BigDecimal for budget**: Monetary precision required; deviates from existing `InventoryItem.unitPrice`
- **PATCH for status**: Semantically correct for partial updates (OrderController uses PUT — future alignment candidate)
- **Campaign-Order FK deferred**: Placeholder method in CampaignService; full implementation in follow-up story to avoid cross-domain schema churn
- **Exception reuse**: No new classes; existing GlobalExceptionHandler covers all cases
